# 🧠 一句话定义：

> **数组**是用来存放**一组相同类型的数据**的“变量集合”。

比如：你想存储 5 个整数，可以写成：

```c
int nums[5]；
```

## ✅ 数组的基本用法

### 1. **定义数组**

```c
int nums[5];        // 定义一个能存5个int的数组
float scores[3];    // 3个 float
char name[10];      // 10个字符
```

### 2. **初始化数组**

```c
int a[5] = {1, 2, 3, 4, 5};         // 完全初始化
int b[5] = {1, 2};                  // 剩下的默认为 0
char c[] = {'H', 'i', '\0'};       // 字符数组（字符串）

// 字符串写法：
char name[] = "Alice";  // 自动加上 '\0'，长度为 6
```

### 3. **访问数组元素**

数组下标从 **0 开始**！

```c
a[0] = 10;            // 修改第一个元素为 10
printf("%d", a[2]);   // 打印第三个元素
```

### 4. **遍历数组**

```c
for (int i = 0; i < 5; i++) {
    printf("%d ", a[i]);
}
```

### 5. **数组作为函数参数**

```c
void print_array(int arr[], int size) {
    for (int i = 0; i < size; i++) {
        printf("%d ", arr[i]);
    }
}

int main() {
    int data[3] = {4, 5, 6};
    print_array(data, 3);  // 把数组传进去
}
```

👉 注意：C 语言函数中传数组，其实是**传地址**，不是复制整个数组！

## 🧊 数组的类型

| 类型   | 示例                 | 说明       |
| ---- | ------------------ | -------- |
| 一维数组 | int a[10];         | 一行数      |
| 字符数组 | char name[20];     | 常用于存字符串  |
| 二维数组 | int mat[3][4];     | 表格、矩阵    |
| 多维数组 | int cube[2][3][4]; | 三维数组，不常用 |

## 📌 二维数组举个例子

```c
int mat[2][3] = {
    {1, 2, 3},
    {4, 5, 6}
};

printf("%d", mat[1][2]);  // 输出6
```

> **你想存 5 个整数，应该写：**
> int nums[5]; ✅
> **不是：** int nums[4]; ❌

## 📌 原因是什么？

虽然数组的下标是从 **0 开始**，但数组的**长度**（即你定义的数字）还是表示你能**存多少个元素**。

### 比如

```c
int nums[5];
```

这表示：你可以存 **5 个整数**，分别是：

| 下标      | 元素        | 说明  |
| ------- | --------- | --- |
| nums[0] | 第 1 个元素   |     |
| nums[1] | 第 2 个元素   |     |
| nums[2] | 第 3 个元素   |     |
| nums[3] | 第 4 个元素   |     |
| nums[4] | 第 5 个元素 ✅ |     |

所以虽然下标最大是 4，但你**一共能存 5 个数**。

## ❌ 如果你写 int nums[4];

你就只能存：

```c
nums[0], nums[1], nums[2], nums[3]  // 共4个元素
```

如果你试图写 nums[4] = 100;

程序不会报错，但运行时可能会**崩溃或数据错乱**（因为越界访问了内存）。

```cpp
char bin[32] = {0};
```

我们来逐步拆解它的意思👇

✅ 一句话解释：

定义一个包含 32 个 char 类型元素的数组 bin，并把所有元素都初始化为 0。

## ⚠️ 如果你不写 = {0} 会怎样？

```c
char bin[32];
```

这个时候 bin 数组的内容是 **未初始化的垃圾值**，可能是乱七八糟的随机内存数据。

如果你用它来判断二进制是否是回文，就可能出错！
